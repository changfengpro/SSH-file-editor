# sfe v0.2.0 第一批功能设计

## 背景

`sfe` 当前已经具备 Vim 风格保存退出、C 语法高亮、当前文件补全、可配置补全快捷键、成对符号补全和 deb 打包能力。v0.2.0 第一批目标是在不引入外部运行时依赖的前提下，让远程 SSH 写 C 代码的体验更接近轻量版 VS Code，同时补齐安装后的系统级使用体验。

## 目标

本批次包含四组能力：

1. 编辑体验基础：行号、状态栏信息、智能缩进、撤销和重做。
2. C 语言体验：扫描当前目录头文件、补全本地头文件、补全头文件里的常见符号、提供简单函数签名提示。
3. VS Code 风格轻量体验：补全与签名提示在 Insert 模式中自然出现，不改变 Vim 风格保存、退出和模式切换。
4. 打包发布体验：deb 安装默认配置示例、`man sfe`、`sfe-upgrade` 自动升级脚本。

## 非目标

以下能力不纳入 v0.2.0 第一批：

- 不接入 `clangd`、`ctags` 或其他第三方服务。
- 不做完整 C 语义分析、类型推导、跳转定义、诊断列表或项目级跨目录索引。
- 不实现完整 Vim 兼容层，例如宏录制、寄存器、visual block、复杂 text object。
- 不在编辑器内做联网升级；升级脚本作为独立命令提供。
- 不要求 Windows 原生 curses 运行支持，仍以 Linux/SSH 终端为主。

## 用户体验

启动后界面显示行号。状态栏展示模式、文件名、修改状态、当前行列、总行数和简短提示。行号区域不参与文本编辑，光标位置仍对应真实文件内容。

Insert 模式下：

- 按 Enter 时继承当前行前导空白。
- 如果光标前最近的有效字符是 `{`，新行额外增加一级缩进。
- Tab 在补全菜单打开时接受补全；菜单未打开时插入配置的缩进宽度，默认 4 个空格。
- 当前文件或同目录头文件提供的符号仍参与补全排序。
- 输入函数名和 `(` 后，如能识别内置或扫描到的函数签名，在底部显示简短签名提示。

Normal 模式下：

- `u` 撤销一次编辑。
- `Ctrl-R` 重做一次编辑。
- 其他已有移动、保存、退出行为保持不变。

安装后：

- `/etc/sfe/config.json` 提供系统默认配置示例。
- 用户配置 `~/.config/sfe/config.json` 优先于系统配置。
- `man sfe` 展示命令用法、快捷键、配置文件、升级方式。
- `sfe-upgrade` 从 GitHub 最新 release 下载 deb 并调用 apt 安装。

## 配置设计

配置读取顺序：

1. 内置默认配置。
2. `/etc/sfe/config.json`，如果存在则覆盖内置默认。
3. `~/.config/sfe/config.json`，如果存在则覆盖系统配置。

新增配置项：

```json
{
  "auto_pair": true,
  "completion_key": "ctrl+space",
  "indent_width": 4,
  "show_line_numbers": true,
  "scan_local_headers": true,
  "signature_help": true
}
```

无效配置值必须被忽略并回退到默认值，避免配置错误导致编辑器无法启动。

## C 语言索引设计

新增轻量头文件扫描器，只扫描当前编辑文件所在目录下的 `.h` 文件，不递归。

扫描内容：

- `#define NAME` 提供宏补全。
- `typedef ... NAME;` 提供 typedef 名称补全。
- `struct NAME` 提供结构体名补全。
- 简单函数声明，例如 `int add(int a, int b);` 提供函数名补全和签名提示。

解析策略以正则和逐行扫描为主，但必须保守：

- 无法可靠识别的声明直接跳过。
- 不修改源文件。
- 不因为单个头文件读取失败而阻止编辑器启动。
- 头文件内容只作为补全和提示来源，不用于语义诊断。

`#include "` 场景补全当前目录 `.h` 文件；`#include <` 场景仍使用内置常见标准头文件。

## 编辑器内部设计

撤销/重做使用快照栈实现第一版：

- 每次实际文本修改前记录快照，包含所有行、光标行列和 dirty 状态。
- 连续普通字符输入可合并为较少快照，避免单字符撤销过碎。
- 保存操作不进入撤销栈。
- 撤销后产生新编辑时清空 redo 栈。

渲染布局：

- 行号宽度根据总行数动态计算，最小宽度为 3。
- 文本绘制起始列 = 行号宽度 + 分隔空格。
- 光标横向位置继续使用 CJK 显示宽度计算，并加上文本起始列。
- 小屏幕下优先保留文本区域，行号宽度不超过必要宽度。

状态栏：

- 保留单行底部状态栏。
- 左侧显示模式和文件名。
- 中间或右侧显示 modified 标记、行列、总行数。
- 临时消息可以覆盖状态栏的一部分，但不能破坏命令模式输入。

签名提示：

- 仅在 Insert 模式显示。
- 根据光标前文本找最近未闭合的 `函数名(`。
- 有匹配签名时显示一行简短提示。
- 不做参数高亮和嵌套调用精确解析。

## 打包设计

deb 包新增文件：

- `/etc/sfe/config.json`：系统默认配置示例。
- `/usr/share/man/man1/sfe.1.gz`：主程序 man page。
- `/usr/bin/sfe-upgrade`：升级脚本。

`sfe-upgrade` 行为：

- 默认查询 `https://api.github.com/repos/changfengpro/SSH-file-editor/releases/latest`。
- 找到 `sfe_*_all.deb` 资产后下载到临时目录。
- 打印版本、下载 URL 和 SHA256。
- 使用 `sudo apt install ./下载文件.deb` 安装。
- 网络失败、没有 release、没有 deb 资产或缺少 `curl` 时给出清晰错误。

所有打包脚本生成的 shell 文件必须使用 Unix 换行，避免 `/bin/sh^M`。

## 测试计划

单元测试覆盖：

- 配置读取顺序和无效值回退。
- 行号布局计算和 CJK 光标列计算。
- 智能缩进：继承缩进、`{` 后增加一级、普通换行保持行为。
- 撤销/重做：单次编辑、连续编辑、redo 清空、保存不入栈。
- 头文件扫描：宏、typedef、struct、简单函数声明、失败文件跳过。
- include 补全：本地头文件和标准头文件区分。
- 签名提示：内置函数和头文件函数。
- deb 打包包含 `/etc/sfe/config.json`、man page、`sfe-upgrade`，并保持 launcher 和脚本 Unix 换行。

手动或远端验证：

- 在 Linux SSH 终端打开 C 文件，确认行号、状态栏、智能缩进、撤销/重做可用。
- 在同目录放置 `.h` 文件，确认 include、宏、struct、函数补全可用。
- 构建 deb 后安装，确认 `sfe`、`man sfe`、`sfe-upgrade --help` 可执行。

## 发布标准

v0.2.0 发布前必须满足：

- 全量单元测试通过。
- `python3 -m py_compile sfe.py sfe_core.py scripts/build_deb.py` 通过。
- 本地 deb 构建成功，包内脚本无 CRLF。
- 远端 `192.168.1.145` 验证通过。
- tag `v0.2.0` 和 GitHub Release 发布成功，release asset 包含 `sfe_0.2.0_all.deb`。
