# v0.5.7 Movement and Status Design

## Goal

Improve long C-file navigation and day-to-day safety feedback without expanding the editor into a larger feature batch.

## Scope

- Normal mode accepts `gg`, `G`, `Ctrl-D`, and `Ctrl-U` using Vim-style movement behavior.
- Quitting with unsaved buffers shows an `Unsaved Buffers` list instead of only a terse `E37` message. Forced quit commands still exit.
- The status bar makes modified files obvious, shows read-only files, and reports elapsed time for `:make` and `:run`.
- README, man page, release notes, version, tag, and Debian assets are updated for `v0.5.7`.

## Behavior

`gg` moves the cursor to the first line and column 0. `G` moves to the last line and clamps the cursor column to that line. `Ctrl-D` and `Ctrl-U` move by half the visible editor height, using a small fallback when tests run without curses.

`:q`, `:quit`, `Ctrl-Q`, and `F10` call the same quit guard. If any buffer is dirty, the editor opens `Unsaved Buffers`, listing current and modified buffers with path labels. `:q!` and `:quit!` still force exit.

The status bar uses `DIRTY` for modified buffers and `RO` for read-only paths. Build and run status messages include elapsed time such as `Build OK in 0.32s`.

## Testing

Unit tests cover each movement command, dirty-buffer quit handling, read-only and dirty status bar markers, and build/run elapsed-time status.
