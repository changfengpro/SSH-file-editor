# SFE v0.5.0 Multi-Buffer Workspace Design

## Goal

Upgrade SFE from a single-file editor with project navigation into a lightweight multi-file workspace. Opening a file from `:e`, `:open`, the project tree, or quick open should switch to an existing buffer or create a new buffer without losing other open files.

## Scope

- Add in-process buffers for multiple open files.
- Add Vim-style buffer commands: `:buffers`, `:bn`, `:bp`, `:bd`.
- Add quick file open via `Ctrl-P` and `:files`.
- Preserve existing save/quit semantics for the current buffer.
- Keep project tree behavior: Enter on a file opens/switches buffer; Enter on a directory toggles expansion.
- Update README, man page, package version, and release notes to `0.5.0`.

## Non-Goals

- Split panes or side-by-side editing.
- Persistent sessions across editor restarts.
- Full Vim buffer command compatibility beyond the commands listed above.
- Background LSP, clangd, or semantic C analysis.

## Behavior

### Buffer Model

Each buffer owns:

- `TextBuffer`
- optional file path
- scroll row offset
- cursor row/column
- diagnostics for that file

The app owns a list of buffers and a current buffer index. Switching buffers restores the cursor and scroll state of the target buffer, refreshes project context, and updates diagnostics.

Opening a path:

- If the path is already open, switch to that buffer.
- If it is not open, save current view state, load the file into a new buffer, remember it in recent files, and switch to it.
- If the current buffer is unnamed and clean, opening the first named file may reuse it to avoid an extra empty buffer.

### Commands

`:buffers`

- Shows one line per buffer.
- Current buffer is marked with `%`.
- Dirty buffers are marked with `+`.
- Unnamed buffers display `[No Name]`.

`:bn` / `:bnext`

- Switches to the next buffer, wrapping at the end.

`:bp` / `:bprevious`

- Switches to the previous buffer, wrapping at the beginning.

`:bd` / `:bdelete`

- Deletes the current buffer if it is clean.
- Refuses to delete a dirty buffer and reports a Vim-style write warning.
- If it deletes the last buffer, it creates a clean unnamed buffer.

`:files`

- Opens a LIST view of project files using fuzzy ranking.
- Pressing Enter on a file in the list opens or switches to that buffer.

`Ctrl-P`

- Same as `:files`.

### Status

The status line should include buffer position, for example `Buf 2/4`, in addition to the current file name, dirty marker, version, and diagnostic count.

## Tests

Add UI-level tests for:

- Opening multiple files creates multiple buffers and keeps previous edits.
- Opening the same file twice switches to the existing buffer.
- `:buffers` lists current and dirty buffers.
- `:bn` and `:bp` wrap correctly.
- `:bd` refuses dirty buffers and deletes clean buffers.
- `Ctrl-P` / `:files` shows project files and Enter opens a buffer.
- Tree file activation opens a buffer and keeps the tree visible.
- Status line includes `Buf current/total`.
