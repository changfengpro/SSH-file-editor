# v0.3.0 C IDE Upgrade Implementation Plan

> **For AI agents:** implement this plan with TDD in small steps. Keep commits focused and verify after each major task.

**Goal:** turn sfe v0.3.0 into a more IDE-like C editor for SSH: project symbols, richer completions, diagnostics, snippets, editor commands, docs, Debian package, tag, and release.

**Architecture:** keep core logic in `sfe_core.py` where it is testable without curses. Keep terminal interaction in `sfe.py`. Avoid new runtime dependencies; use Python standard library only.

**Tech stack:** Python 3.10+, curses, unittest, custom Debian package builder.

---

### Task 1: Project Symbol Index

**Files:**
- Modify: `sfe_core.py`
- Tests: `tests/test_sfe_core.py`

- [ ] Add `ProjectSymbol` and `ProjectIndex` dataclasses.
- [ ] Add `ProjectScanner.scan(root)` to scan `.c` and `.h` files recursively with a small ignore list.
- [ ] Detect functions, macros, structs, typedefs, and simple globals with file, row, column, kind, and detail.
- [ ] Add tests with temporary projects and expected symbol locations.

### Task 2: Richer Completion Items and Snippets

**Files:**
- Modify: `sfe_core.py`
- Tests: `tests/test_sfe_core.py`

- [ ] Extend completion items with `source`, `detail`, and optional snippet insertion text.
- [ ] Add C snippets for `main`, `for`, `if`, and `switch`.
- [ ] Include project symbols in suggestions.
- [ ] Improve ranking so exact/prefix C keywords and snippets beat fuzzy project symbols when appropriate.

### Task 3: C Diagnostics

**Files:**
- Modify: `sfe_core.py`
- Tests: `tests/test_sfe_core.py`

- [ ] Add `Diagnostic` dataclass and `CDiagnosticEngine`.
- [ ] Detect unmatched brackets, unterminated strings, duplicate includes, and simple missing semicolons after declarations or assignments.
- [ ] Return stable row/column/message/severity data for editor display and navigation.

### Task 4: Editor Commands and Navigation

**Files:**
- Modify: `sfe.py`, `sfe_core.py`
- Tests: `tests/test_sfe_core.py`, `tests/test_sfe_ui.py`

- [ ] Support `:goto <line>`.
- [ ] Support `:symbols` and `:diag` as editor list modes.
- [ ] Support `:help` in-app help text.
- [ ] Support `:e <file>` and `:w <file>`.
- [ ] Support `:set auto_pair on/off`, `:set completion_key <key>`, and `:set number on/off`, writing user config when possible.
- [ ] Support jump-to-definition through project index and jump-back stack.

### Task 5: UI Integration

**Files:**
- Modify: `sfe.py`
- Tests: `tests/test_sfe_ui.py`

- [ ] Show richer completion rows with text, kind, source, and detail.
- [ ] Insert snippets with multiline text and useful cursor placement.
- [ ] Show status line with version and diagnostic count.
- [ ] Add diagnostic navigation helpers.

### Task 6: Documentation, Packaging, Release

**Files:**
- Modify: `VERSION`, `README.md`, `packaging/debian/sfe.1`
- Verify: `scripts/build_deb.py`, `tests/test_packaging.py`

- [ ] Set version to `0.3.0`.
- [ ] Document new commands, diagnostics, project symbols, snippets, and release package names.
- [ ] Build `dist/sfe_0.3.0_all.deb`.
- [ ] Verify tests, py_compile, deb contents, line endings, and release assets.
- [ ] Merge to main, tag `v0.3.0`, push, create GitHub Release, upload `sfe_0.3.0_all.deb` and `sfe_latest_all.deb`.
