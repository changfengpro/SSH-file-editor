# SFE v0.4.0 项目工作流实现计划

> **面向 AI 代理的工作者：** 必须使用 TDD。每个行为先写失败测试，再实现最小代码让测试通过，最后运行完整验证。

**目标：** 让 SFE 从单文件 C 编辑器升级为轻量项目编辑器，支持项目文件浏览、模糊打开、最近文件、构建运行和编译错误导航。

**架构：** 纯逻辑继续放在 `sfe_core.py`，包括项目根识别、文件扫描、模糊匹配、最近文件和构建输出解析。`sfe.py` 只负责 Vim 风格命令、curses 列表显示、构建进程调用和光标跳转。文档、man page、默认配置和 deb 打包同步升级到 `0.4.0`。

**技术栈：** Python 标准库、`unittest`、curses、Debian ar/tar/gzip 打包脚本、GitHub Release API。
